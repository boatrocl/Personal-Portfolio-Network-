## 📖 ภาพรวมโครงการ

**Tastes Through Network (TTN)** คือระบบเครือข่ายที่ออกแบบขึ้นใหม่สำหรับการส่งข้อมูลรสชาติในรูปแบบดิจิทัล โดยแปลงรสชาติ (หวาน เค็ม เปรี้ยว ขม เผ็ด) กลิ่น และภาพออกมาเป็น data packet แล้วส่งผ่านเครือข่ายไปยังอุปกรณ์ที่ผู้รับสวมใส่ เพื่อกระตุ้นประสาทสัมผัสให้รู้สึกเหมือนกำลังรับประทานอาหารจริง

**ตัวอย่าง:** กินข้าวเปล่า แต่รู้สึกรสชาติเหมือนแกงเขียวหวาน 🍛

## 🏛️ สถาปัตยกรรมเครือข่าย (5 ชั้น)

TTN ออกแบบตามโมเดล 5 ชั้น ดังนี้:

```
┌─────────────────────────────────────┐
│  5. APPLICATION LAYER               │
│     Phantom Banquet App             │
│     ให้ผู้ใช้เลือกเมนูอาหาร         │
└──────────────────┬──────────────────┘
                   ↓
┌─────────────────────────────────────┐
│  4. TRANSPORT LAYER                 │
│     NAP: Neural Auth Protocol       │
│     ใช้ UDP — เร็ว ไม่รอการตอบรับ    │
└──────────────────┬──────────────────┘
                   ↓
┌─────────────────────────────────────┐
│  3. NETWORK LAYER                   │
│     TRP: Temporal Routing Protocol  │
│     Cost = 0.6×เวลา + 0.3×คุณภาพ + 0.1×ความเสี่ยง │
│     อัพเดทเส้นทางทุก 10 วินาที      │
└──────────────────┬──────────────────┘
                   ↓
┌─────────────────────────────────────┐
│  2. DATA LINK LAYER                 │
│     BSAP: Bio-Sensor Addressing     │
│     ที่อยู่อุปกรณ์ 24 ไบต์           │
│     ยืนยันตัวตนด้วย Neural Signature │
└──────────────────┬──────────────────┘
                   ↓
┌─────────────────────────────────────┐
│  1. PHYSICAL LAYER                  │
│     NPI: Neuro-Palate Interface     │
│     อุปกรณ์สวมใส่ (ช้อน, แว่น AR)   │
│     กระตุ้นรสชาติด้วยไฟฟ้าอ่อนๆ     │
│     ฉีดกลิ่นด้วยอัลตร้าซาวด์        │
└─────────────────────────────────────┘
```

### Custom Protocols ที่พัฒนาขึ้น

| Protocol | ชื่อเต็ม | หน้าที่ |
|----------|----------|---------|
| **NAP** | Neural Auth Protocol | ยืนยันตัวตน + ส่งข้อมูลแบบ UDP |
| **TRP** | Temporal Routing Protocol | หาเส้นทางที่เร็วที่สุด |
| **BSAP** | Bio-Sensor Addressing Protocol | กำหนดที่อยู่อุปกรณ์ชีวภาพ |
| **NPI** | Neuro-Palate Interface | ฮาร์ดแวร์กระตุ้นประสาทสัมผัส |

## 🔄 ตัวอย่างการทำงาน

1. ผู้ใช้เปิดแอพ **Phantom Banquet** เลือกเมนู "แกงเขียวหวาน" (ชั้น 5)
2. ข้อมูลรสชาติถูกแปลงเป็น packet และส่งแบบ UDP (ชั้น 4)
3. ระบบ TRP หาเส้นทางที่มี latency ต่ำสุด (ชั้น 3)
4. BSAP เพิ่มที่อยู่ผู้ส่ง-ผู้รับ (ชั้น 2)
5. อุปกรณ์ NPI กระตุ้นลิ้น จมูก และตา (ชั้น 1)

### TasteProfile Format

```json
{
  "รส": {

  },

    "ตะไคร้": 0.6
  },
  "ภาพ": {
    "สี": "เขียว",
    "หน้าตา": "ใบมะกรูด"
  }
}
```
